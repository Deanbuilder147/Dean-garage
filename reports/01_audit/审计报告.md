## 🚨 关键问题 (生产前必须修复)

### 1️⃣ 安全问题 🔴 高优先级

#### 问题 1.1: 硬编码的秘密密钥
**位置:** `.env`, `docker-compose.yml`
```bash
JWT_SECRET=mecha-battle-secret-2026  # 太弱了！
POSTGRES_PASSWORD=mecha_secure_password  # 可预测
```

**风险:** 攻击者可以伪造 JWT 令牌，盗取用户账户

**修复方案:**
- 生成强随机密钥 (32+ 字符)
- 使用环境变量，不要提交到 git
- 实现密钥轮换机制

#### 问题 1.2: 缺少输入验证
**位置:** 所有服务的路由文件
```javascript
// 没有验证用户输入
app.post('/api/auth/register', (req, res) => {
  const { username, password } = req.body; // 不安全！
});
```

**风险:** SQL 注入、XSS 攻击、数据越界

**修复方案:**
- 添加验证库 (Zod, Joi, Yup)
- 过滤所有输入数据
- 实现速率限制

#### 问题 1.3: 关键路由缺少认证
**位置:** `services/combat-service/src/middleware/auth.js:147`
```javascript
// TODO: 需要从房间服务获取房间创建者信息
```
部分关键操作没有验证权限

**修复方案:** 
- 为所有受保护路由添加认证中间件
- 实现房间创建者权限检查

#### 问题 1.4: CORS 配置过于宽松
**位置:** 所有服务的 app.js
```javascript
app.use(cors()); // 允许所有来源 - 危险！
```

**修复方案:** 生产环境限制为特定域名

### 3️⃣ 数据库问题 🟠 高优先级

#### 问题 3.1: 使用内存数据库
**位置:** 所有服务使用 `sql.js` (SQLite 内存模式)

**问题:**
- 服务重启后数据丢失
- 无法持久化存储
- 没有数据迁移机制

**修复方案:**
- 切换到 PostgreSQL (已在 docker-compose 中配置)
- 添加数据迁移系统 (Knex.js 或 Prisma)
- 创建备份策略

#### 问题 3.2: 缺少数据库索引
**影响:** 数据量增长后查询变慢

**修复方案:** 为以下字段添加索引:
- `users.username`
- `units.user_id`
- `battles.id`
- `battle_moves.battle_id`

#### 问题 3.3: 缺少连接池
**问题:** 每个请求都创建新数据库连接

**修复方案:** 实现连接池

### 5️⃣ 性能问题 🟡 中优先级

#### 问题 5.1: 没有缓存
**问题:** 每个请求都查询数据库

**修复方案:**
- 添加 Redis 缓存
- 缓存频繁访问的数据 (机甲、战场)
- 实现缓存失效机制

#### 问题 5.2: 没有速率限制
**问题:** API 容易被 DDoS 攻击

**修复方案:**
```javascript
// 每 15 分钟每 IP 最多 100 个请求
const limiter = rateLimit({
  windowMs: 15 * 60 * 1000,
  max: 100
});
app.use('/api/', limiter);
```

**位置:** `frontend/src/assets/`

**修复方案:**

- 实现懒加载
