### 修复 #4: 认证绕过漏洞

**问题：** map-service 在生产环境下允许无 token 访问

**修复内容：**

**文件：** `services/map-service/src/routes/battlefields.js`

**修改前：**
```javascript
// 开发环境：如果没有 token，使用测试用户 ID=1
if (!authHeader) {
  req.user = { userId: 1, username: 'test' };
  return next();
}
```

**修改后：**
```javascript
// 生产环境：必须提供 token
// 开发环境：允许无 token 访问测试用户（向后兼容）
if (!authHeader) {
  if (process.env.NODE_ENV === 'production') {
    return res.status(401).json({ error: '未授权访问' });
  }
  // 开发环境保持 test 用户兼容
  req.user = { userId: 1, username: 'test' };
  return next();
}
```

**同时移除了 catch 块中的回退逻辑**，确保 Token 无效时正确返回错误。

**修改的文件：**
- ✅ services/map-service/src/routes/battlefields.js

**验证方法：**
```bash
# 生产环境无 token 访问应返回 401
NODE_ENV=production curl -X GET http://localhost:3003/api/battlefields
# 应返回：{"error":"未授权访问"}

# 开发环境应允许无 token
NODE_ENV=development curl -X GET http://localhost:3003/api/battlefields
# 应返回数据
```

### 修复 #5: JWT 密钥弱回退

**问题：** 8 个文件中使用 `|| 'default-secret'` 弱回退

**修复内容：**

所有 JWT_SECRET 使用处改为强制要求环境变量：

```javascript
// 修改前：
const JWT_SECRET = process.env.JWT_SECRET || 'mecha-battle-auth-secret-key';

// 修改后：
// 强制要求 JWT_SECRET
const JWT_SECRET = process.env.JWT_SECRET;
if (!JWT_SECRET) {
  throw new Error('[配置错误] JWT_SECRET 环境变量必须设置！');
}
```

### 修复 #6: DB 存储在/tmp 目录

**问题：** combat-service 使用 `/tmp` 存储数据库，重启后数据丢失

**修复内容：**

**文件：** `services/combat-service/src/database/db.js`

**修改前：**
```javascript
const DB_DIR = '/tmp';
const DB_PATH = path.join(DB_DIR, 'combat.db');
```

**修改后：**
```javascript
// 使用数据目录（避免/tmp 被清空）
const DB_DIR = process.env.DB_PATH || path.join(__dirname, '../../data');
const DB_PATH = path.join(DB_DIR, 'combat.db');

// 在 initializeDatabase 中：
// 确保数据目录存在并设置正确权限
if (!fs.existsSync(DB_DIR)) {
  fs.mkdirSync(DB_DIR, { recursive: true, mode: 0o700 });
}
```

**修改的文件：**
- ✅ services/combat-service/src/database/db.js

**验证方法：**
```bash
# 检查无/tmp 路径
grep -r "DB_DIR.*=.*'/tmp'" services/*/src/
# 应无输出

# 检查数据目录权限
ls -la /app/data  # 应为 700 权限
```
