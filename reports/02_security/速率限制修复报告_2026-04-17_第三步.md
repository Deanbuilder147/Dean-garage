## 🔧 修复详情

### 速率限制配置 (所有服务)

**安装的依赖：**
```bash
npm install express-rate-limit
```

**标准速率限制器配置：**
```javascript
const limiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 分钟窗口
  max: 100, // 每个 IP 最多 100 次请求
  message: { error: '请求过多，请稍后重试', retryAfter: 900 },
  standardHeaders: true,
  legacyHeaders: false,
  skip: (req) => req.path === '/health' // 健康检查不限速
});

app.use('/api/', limiter);
```

## 🎯 限流特性

### 通用特性

- ✅ **健康检查排除** - `/health` 端点不限速，确保监控正常
- ✅ **标准 HTTP 头** - 使用 `standardHeaders: true` 返回限流信息
- ✅ **友好错误消息** - 中文错误提示 + 重试时间
- ✅ **基于 IP 限流** - 每个 IP 地址独立计数

### 特殊保护

- ✅ **登录接口保护** - auth-service 的登录接口有额外限流 (10 次/15 分钟)
- ✅ **防止暴力破解** - 有效阻止密码爆破攻击
- ✅ **防止 DDoS** - 限制单 IP 请求频率
