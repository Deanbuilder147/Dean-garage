# Button 组件事件传递修复报告
**日期**: 2026-04-17 23:05  
**问题**: 点击"选择战场"按钮时报错，页面跳转到错误路径  
**根本原因**: Button 组件未传递事件对象 `$event`

## 问题分析

### 错误信息
```
Uncaught TypeError: Cannot read properties of undefined (reading 'stopPropagation')
    at stop (chunk-A5RM4ZU6.js?v=3fa01771:12832:18)
    at emit (chunk-A5RM4ZU6.js?v=3fa01771:6853:5)
    at Button.vue:5:13
```

### 问题代码
**文件**: `frontend/src/components/ui/Button.vue` 第 5 行

```vue
<!-- ❌ 错误：没有传递 $event -->
<button @click="$emit('click')">
```

### 触发场景
在 `BattlefieldSelector.vue` 中，按钮使用了 `.stop` 修饰符：
```vue
<Button @click.stop="confirmSelection(bf)">
```

`.stop` 修饰符会在事件处理器中调用 `event.stopPropagation()`，但 Button 组件的 `$emit('click')` 没有传递事件对象，导致父组件接收到的事件参数是 `undefined`。

## 修复方案

### 修改内容
**文件**: `frontend/src/components/ui/Button.vue`

```diff
-    @click="$emit('click')"
+    @click="$emit('click', $event)"
```

### 修复原理
1. `$event` 是 Vue 模板中的特殊变量，代表当前 DOM 事件对象
2. 通过 `$emit('click', $event)` 将完整的事件对象传递给父组件
3. 父组件使用 `.stop` 修饰符时可以正常调用 `stopPropagation()`

## 为什么之前会跳转到 q1 文件？

这是一个**巧合**：
1. 前端代码抛出 JavaScript 错误
2. 错误导致点击事件处理失败
3. 可能触发了某些未处理的错误边界或默认行为
4. 浏览器可能尝试加载错误的 URL

实际上并没有真正的"跳转到 q1"，而是：
- JavaScript 错误阻止了正常的 Vue Router 导航
- 用户可能手动打开了错误日志文件 q1

## 后续建议
