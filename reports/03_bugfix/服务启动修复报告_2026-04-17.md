## 📋 问题诊断

根据 `dialoguenote.md` 文件的记录，之前发现了 **comm-service** 存在 `dotenv` 加载顺序问题。经过全面检查，发现这是一个**系统性问题**，影响多个微服务。

### 🔍 问题根源

在 ES 模块中，`import` 语句会在模块代码执行之前被评估。如果子模块在顶部直接读取 `process.env.JWT_SECRET` 并抛出错误，而 `dotenv.config()` 在父模块中调用，则子模块会在环境变量加载之前就抛出异常。

**错误示例：**
```javascript
// socketService.js 或 config/index.js
const JWT_SECRET = process.env.JWT_SECRET;
if (!JWT_SECRET) {
  throw new Error('[配置错误] JWT_SECRET 环境变量必须设置！');
}
```

这段代码在模块加载时立即执行，此时 `dotenv.config()` 还未被调用。

## 📝 技术总结

**问题类型：** ES 模块加载顺序问题  
**影响范围：** 4 个微服务  
**修复方案：** 在每个需要读取环境变量的模块中显式调用 `dotenv.config()`  
**修复时间：** 约 15 分钟  

**关键教训：**
- 在 ES 模块中，`import` 语句在代码执行前评估
- 如果子模块依赖环境变量，必须在子模块内部调用 `dotenv.config()`
- 不能依赖父模块的 `dotenv.config()` 调用
