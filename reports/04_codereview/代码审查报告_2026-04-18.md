### ⚠️ 关键问题 (Critical Issues)

#### 1. 数据库持久化风险 🔴 严重
**问题：** Combat-service 使用 sql.js 内存数据库，数据存储在 `/tmp` 目录

**影响：**
- 服务重启后战斗数据丢失
- `/tmp` 可能被系统定期清理
- 无法支持跨服战斗历史查询

**当前代码：**
```javascript
// services/combat-service/src/database/db.js:14
const DB_PATH = process.env.DB_PATH || path.join(__dirname, '../../data');
```

**修复建议：**
```javascript
// 1. 强制使用持久化目录
const DB_PATH = process.env.DB_PATH || '/var/lib/mecha-battle/combat.db';

// 2. 迁移到 PostgreSQL (已有配置)
// 修改 docker-compose.yml 启用 PostgreSQL 连接
```

**优先级：** P0 (立即修复)

#### 2. 硬编码敏感信息 🟠 高
**问题：** `.env` 文件中存在部分硬编码密钥

**示例：**
```bash
# .env:9
POSTGRES_PASSWORD=ma4jQG...o5g=  # ❌ 不应出现在版本控制

# .env:14
JWT_SECRET=mecha-...2026  # ❌ 弱密钥模式
```

**风险：**
- 数据库密码泄露
- JWT Token 可被伪造
- 生产环境安全风险

**修复建议：**
```bash
# 1. 生成强密钥
openssl rand -base64 32  # JWT_SECRET
openssl rand -hex 32     # POSTGRES_PASSWORD

# 2. 确保 .env 在 .gitignore 中
echo ".env" >> .gitignore

# 3. 使用 .env.example 作为模板
cp .env .env.example
# 编辑 .env.example，删除真实密钥，保留占位符
```

**优先级：** P0 (立即修复)

#### 3. 日志泄露风险 🟠 高
**问题：** 生产环境未过滤敏感日志

**示例：**
```javascript
// services/online-battle-service/src/index.js
console.log(`User ${username} authenticated via WebSocket`);  // ❌ 泄露用户活动
console.log(`User ${username} joined room ${roomId}`);        // ❌ 泄露房间信息
```

**影响：**
- 用户行为模式可被追踪
- 房间对战信息泄露
- 日志文件可能包含敏感数据

**修复建议：**
```javascript
// 1. 安装 Winston
npm install winston

// 2. 替换 console.log
import winston from 'winston';

const logger = winston.createLogger({
  level: process.env.LOG_LEVEL || 'info',
  format: winston.format.json(),
  transports: [
    new winston.transports.File({ filename: 'logs/app.log' })
  ]
});

// 开发环境输出控制台
if (process.env.NODE_ENV !== 'production') {
  logger.add(new winston.transports.Console());
}

// 3. 使用 logger 替代 console
logger.info('User authenticated', { userId: decoded.userId });  // ✅ 结构化日志
```

**优先级：** P1 (本周内修复)

#### 5. 服务间通信缺少认证 🟡 中
**问题：** 微服务间 API 调用无认证机制

**示例：**
```javascript
// frontend/src/api/client.js
const apiClient = axios.create({
  baseURL: '/api',
  // ❌ 无服务间认证
});
```

**风险：**
- 内部服务可被伪造请求
- 跨服攻击可能
- 权限提升漏洞

**修复建议：**
```javascript
// 1. 添加服务间 JWT
// services/auth-service/src/middleware/serviceAuth.js
export const verifyServiceToken = (req, res, next) => {
  const serviceToken = req.headers['x-service-token'];
  
  if (!serviceToken || serviceToken !== process.env.SERVICE_SECRET) {
    return res.status(401).json({ error: 'Service unauthorized' });
  }
  
  next();
};

// 2. 在所有内部 API 端点应用
app.use('/api/internal', verifyServiceToken, internalRoutes);
```

**优先级：** P2 (下迭代)

## 📝 代码规范建议

### 1. 命名规范

**当前问题：**
```javascript
// ❌ 混合命名风格
const battle_sessions = db.all('SELECT * FROM battle_sessions');  // snake_case
const currentFaction = 'earth';  // camelCase
```

**建议：**
```javascript
// ✅ JavaScript: camelCase
const battleSessions = db.all('SELECT * FROM battle_sessions');
const currentFaction = 'earth';

// ✅ SQL: snake_case (保持不变)
// ✅ 组件：PascalCase
// ✅ 常量：UPPER_SNAKE_CASE
```

### 2. 错误处理

**当前问题：**
```javascript
// ❌ 吞掉错误
try {
  const result = db.run(sql);
} catch (error) {
  console.error('Error:', error);  // 仅日志，不处理
}
```

**建议：**
```javascript
// ✅ 向上传播或恢复
try {
  const result = db.run(sql);
  return result;
} catch (error) {
  logger.error('Database operation failed', { sql, error: error.message });
  throw new DatabaseError(`Query failed: ${error.message}`);
}
```
