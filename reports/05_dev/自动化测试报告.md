**影响的服务:** auth-service, hangar-service, map-service

**问题:** 
- 测试脚本期望 `/api/auth/health`
- 实际可能是 `/health` 或其他路径

**期望行为:** 所有服务应该有统一的健康检查端点路径

**建议修复:**

```javascript
// 所有服务统一使用
app.get('/health', (req, res) => {
  res.json({ status: 'ok', service: 'xxx', timestamp: Date.now() });
});
```

### Bug #3: 用户名验证错误消息不够友好 🟡 中优先级

**测试场景:** 用户注册

**实际错误:** `{"error":"用户名长度需在 3-20 个字符之间"}`

**问题:** 虽然是中文，但对于国际用户不友好

**建议:** 
```javascript
{
  "error": "用户名长度需在 3-20 个字符之间",
  "error_en": "Username must be between 3 and 20 characters",
  "field": "username",
  "validation": { "min": 3, "max": 20 }
}
```

---

### Bug #4: 登录失败没有详细错误信息 🟡 中优先级

**测试场景:** 登录失败

**问题:** 只返回"登录失败"，没有说明是用户名错误还是密码错误

**安全风险:** 详细错误可能泄露用户信息（这是对的！）

**建议:** 保持模糊但添加错误代码
```javascript
{
  "error": "认证失败",
  "code": "AUTH_FAILED"
}
```

---

### Bug #5: 缺少请求追踪 ID 🟢 低优先级
