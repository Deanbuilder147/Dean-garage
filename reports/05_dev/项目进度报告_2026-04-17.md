## ✅ 已完成的安全加固 (2026-04-17)

### 第一阶段：核心安全问题修复

#### 1. 硬编码密钥 ✅
- **状态:** 已修复
- **措施:**
  - 生成强 JWT_SECRET (64 字符随机)
  - PostgreSQL 密码使用环境变量
  - 创建 `.env` 文件包含安全随机值
  - `docker-compose.yml` 更新为使用 `${VARIABLE}` 语法
  - 提供 `.env.example` 模板

#### 2. 输入验证缺失 ✅
- **状态:** 已修复
- **措施:**
  - 6 个服务全部安装 Zod 验证库 (auth, hangar, map, combat, comm, online-battle)
  - auth-service: 为 `/register` 和 `/login` 创建 registerSchema 和 loginSchema
  - combat-service: 为 POST `/battles` 创建 createBattleSchema
  - 详细的验证错误消息返回给客户端
  - Zod 错误包含字段名和验证消息

#### 3. 关键路由缺少认证 ✅
- **状态:** 已修复
- **位置:** `map-service/src/routes/battlefields.js`
- **措施:**
  - 生产环境现在需要有效的 JWT 令牌
  - 开发环境保持向后兼容
  - 移除 catch 块回退逻辑

#### 4. CORS 配置错误 ✅
- **状态:** 已修复
- **位置:** 全部 6 个服务
- **措施:**
  - 所有服务现在使用 `ALLOWED_ORIGINS` 环境变量
  - 默认允许来源：`http://localhost:8081,http://localhost:8080,https://yourdomain.com`
  - CORS 配置包含 credentials 和明确的方法
  - `.env` 和 `.env.example` 已更新 `ALLOWED_ORIGINS` 变量

#### 5. JWT Secret 弱回退 ✅
- **状态:** 已修复
- **位置:** 9 个服务文件
- **措施:**
  - 所有 `|| 'default-secret'` 模式已移除
  - 如果未设置 JWT_SECRET，服务将抛出错误

#### 6. 数据库存储在 /tmp ✅
- **状态:** 已修复
- **位置:** `combat-service/src/database/db.js`
- **措施:**
  - 从 `/tmp` 改为 `data/` 目录
  - 目录以 0o700 权限创建
  - 数据在重启后持久化

### 第二阶段：速率限制 (2026-04-17)

#### 7. 速率限制实现 ✅

### 三大阵营

| 阵营 | 特点 | 核心单位 |
|------|------|----------|
| 地球联合 | 均衡攻防，护甲优秀 | 重装机甲 |
| 拜隆 | 火力强大，远程优势 | 炮击机甲 |
| 马克西翁 | 奇袭系统，隐身突袭 | 隐形机甲 |

### 词条系统 (10+ 可组合词条)

| 词条 | 触发阶段 | 效果 |
|------|----------|------|
| 联防 | movement_check | 阻挡非同阵营穿越 |
| 再动 | on_kill | 击杀后额外回合 |
| 幸运 | turn_start | 额外行动判定 |
| 援助 | on_ally_attacked | 分担伤害/反击 |
| 斩杀 | post_damage | 掷骰≥血量斩杀 |
| 抢夺 | post_damage | 掷骰>3 获得武器 |
| 专注射击 | pre_attack | 远程伤害 +3~+5 |
| 抗性 | on_damage_taken | 伤害 -2 |
| 决斗 | pre_attack | 双方掷骰比大小 |
| 空投 | round_start | 生成武器防具 |
