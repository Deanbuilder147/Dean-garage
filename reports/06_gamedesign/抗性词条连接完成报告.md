## 完成的工作

### 1. equipManager.cjs - 添加抗性类型支持
**文件**: `/services/combat-service/src/services/combatCore/equipManager.cjs`

**新增方法**:
- `getArmorResistanceType(unit)` - 获取单位防具的抗性类型（'kinetic' 或 'energy'）
- `armorMatchesDamageType(unit, attackDamageType)` - 检查防具是否匹配攻击伤害类型

**更新方法**:
- `getArmorDefense(unit)` - 现在包含 `resistance_type` 字段在返回的 sources 中

**逻辑**:
- 优先返回左手防具的抗性类型
- 如果左手无防具，返回右手防具的抗性类型
- 默认抗性类型为 'kinetic'（实体伤害）

### 2. tagProcessor.cjs - 更新防具检查方法
**文件**: `/services/combat-service/src/services/combatCore/tagProcessor.cjs`

**更新方法**:
- `hasArmor(unit)` - 现在检查 `left_hand_type` 和 `right_hand_type` 是否为 'armor'
- `getArmorResistanceType(unit)` - 调用 EquipmentManager 的方法
- `armorMatchesDamageType(unit, attackDamageType)` - 新增方法，调用 EquipmentManager

**检查项映射** (resolveCheck):
- `has_armor` - 检查目标是否有防具
- `armor_resistance_type` - 获取目标防具的抗性类型
- `attack_damage_type` - 从上下文获取攻击伤害类型

### 3. combatIntegrator.cjs - 修复上下文变量名
**文件**: `/services/combat-service/src/services/combatCore/combatIntegrator.cjs`

**修复**:
- 将 `damageType` 重命名为 `attackDamageType` (第 216 行)
- 确保与 tagProcessor 的检查项名称匹配

**逻辑**:
```javascript
attackDamageType: attackType === 'melee' ? 'kinetic' : 'energy'
```
- 近战攻击 (melee) = 'kinetic' (实体伤害)
- 远程攻击 (ranged) = 'energy' (能量伤害)

### 4. resistance.cjs - 词条定义验证
**文件**: `/services/combat-service/src/services/combatCore/tags/resistance.cjs`

**现有配置** (无需修改):
```javascript
conditions: {
  required: [
    { check: 'has_armor', value: true },
    { check: 'armor_resistance_type', ref: 'attack_damage_type', operator: '==' }
  ],
  armor_declaration: {
    types: ['kinetic', 'energy'],
    value: 2
  }
}
```

**触发条件**:
1. 目标必须有防具 (`has_armor === true`)
2. 防具抗性类型必须匹配攻击伤害类型 (`armor_resistance_type === attack_damage_type`)

**效果**:
- 伤害减免 2 点 (`damage_reduction: 2`)
- 在伤害计算前应用 (`subtract_before_armor`)

## 数据流

### 防具数据结构
```javascript
{
  left_hand_type: 'armor',
  left_hand_defense: 3,
  left_hand_durability: 0,
  left_hand_resistance_type: 'kinetic' // 新增字段
}
```

### 触发流程
1. **攻击发起** → combatIntegrator.executeAttack()
2. **设置上下文** → `attackDamageType = 'kinetic' | 'energy'`
3. **造成伤害** → damagePipe.calculate()
4. **触发受到伤害钩子** → `on_damage_taken` 阶段
5. **处理抗性词条** → tagProcessor.processPhase('on_damage_taken', context)
6. **检查条件**:
   - `has_armor(target)` → true/false
   - `getArmorResistanceType(target)` → 'kinetic' | 'energy'
   - `attack_damage_type` → 从上下文获取
7. **条件匹配** → 如果抗性类型匹配，触发伤害减免
8. **应用效果** → 伤害 -2

## 测试场景

### 场景 1: 实体伤害抗性 vs 近战攻击
- 防御方：左手防具，`resistance_type: 'kinetic'`
- 攻击方：近战攻击 (`melee` → `kinetic`)
- 结果：✅ 抗性触发，伤害 -2

### 场景 2: 实体伤害抗性 vs 远程攻击
- 防御方：左手防具，`resistance_type: 'kinetic'`
- 攻击方：远程攻击 (`ranged` → `energy`)
- 结果：❌ 抗性不匹配，无减免

### 场景 3: 能量伤害抗性 vs 远程攻击
- 防御方：左手防具，`resistance_type: 'energy'`
- 攻击方：远程攻击 (`ranged` → `energy`)
- 结果：✅ 抗性触发，伤害 -2

### 场景 4: 无防具
- 防御方：无防具
- 攻击方：任何攻击
- 结果：❌ 无防具，抗性不触发
